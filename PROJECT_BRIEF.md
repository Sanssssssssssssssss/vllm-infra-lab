# PROJECT_BRIEF

## 项目名称
本地大模型部署与 Agent 推理服务

## 项目目标
在一台笔记本电脑上完成本地大模型部署与 API 服务实现，使另一台电脑上的 agent 系统可以通过网络调用该模型进行推理。

当前首要目标：
- 跑通 `Qwen3-8B` 的本地推理服务
- 提供稳定、可复用的 API 接口
- 建立可重复启动、可观测、可继续优化的基础工程

中期探索目标：
- AWQ 量化
- 蒸馏相关实验
- LoRA 微调/适配
- tracing / 调试 / 性能分析
- 系统化理解大模型推理优化与 agent 系统集成

## 背景
用户当前硬件环境：
- CPU：Intel Core Ultra 9 185H
- GPU：NVIDIA RTX 4070 Laptop
- GPU 专享显存：8 GB
- GPU 共享内存：18 GB
- 系统内存：32 GB

项目目录：`E:\GPTProject2\vLLM`

## 项目范围
### V1 范围
- 明确本地部署方案与运行环境
- 完成单模型部署（目标模型为 `Qwen3-8B`）
- 提供基础推理 API
- 支持从另一台电脑通过网络访问
- 提供启动、停止、健康检查、基础验证方法
- 形成后续量化、微调、性能优化的工程基线

### 后续范围
- AWQ 量化方案验证
- 推理吞吐/延迟基准测试
- agent tracing 接入
- LoRA 训练与加载
- 蒸馏实验链路

## 成功标准
- 另一台电脑可通过 API 成功调用本机模型并获得推理结果
- 模型服务可以稳定启动、关闭、重启
- 关键配置、启动方式、验证步骤被文档化
- 有明确的性能基线，可用于后续优化对比
- 项目状态与决策可通过记忆文件持续追踪

## 非目标
- 首阶段不追求多模型并发管理
- 首阶段不追求公网暴露或生产级多租户能力
- 首阶段不直接展开复杂训练平台建设
- 首阶段不在未建立基线前同时推进量化、微调、蒸馏

## 约束
- 必须使用项目文档作为长期记忆，而非仅依赖聊天上下文
- 方案需优先简单、稳健、可维护
- 实现必须与当前硬件资源匹配
- 不擅自扩展需求，不在没有依据时引入复杂架构

## 当前已知假设
- 当前项目优先目标是“本地部署 + 可远程调用 API”，而不是训练优先
- 目录名 `vLLM` 表示首选会优先评估 vLLM 路线
- 首版本默认先做单用户、单机宿主、局域网访问
- 首版本可接受先建立功能闭环，再逐步追求极限性能

## 关键未知项
- 宿主运行环境是否为 Windows 原生、WSL2 还是其他 Linux 环境
- 目标 API 是否要求完全兼容 OpenAI API
- 另一台电脑的调用场景是局域网、直连还是更复杂网络环境
- 对吞吐、首 token 延迟、并发数的最低要求尚未明确
- 未来 AWQ / LoRA / tracing 预计采用哪些具体工具链尚未确定

## 里程碑草案
1. 项目初始化与需求澄清
2. 基础部署方案确定
3. Qwen3-8B 本地推理服务跑通
4. 远程调用验证通过
5. 性能基线与优化入口建立
6. 量化/微调/追踪实验阶段
